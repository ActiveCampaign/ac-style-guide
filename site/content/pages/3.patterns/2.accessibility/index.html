Color Contrast

Throughout our products our primary focus is adhering to WCAG AA (4.5:1) with some exceptions. For more information on how to use color to meet this standard, refer to our colors page for a complete guide.

Low Vision / No Vision

Ensuring those with more severe visual impairments have complete control and access to our products, the following should be considered in any case:

  • If using an image, provide an adequate ALT attribute for description and include an ARIA attribute for screen readers when necessary.
  • Controls should be clearly identifiable via all their states and accessible for screen readers. They should be easy to navigate using keyboard controls and colors should adhere to our color usage guide.
  • Controls should always be consistent and predictable.
  • Text sizes and line heights should be sized appropriately and have the ability to be adjusted if needed by the user.
  • Text colors should be chosen carefully. Rever to our typography color guide for more information.
  • Navigational elements should be clearly identifiable and understandable by a screen reader. They should also be easy to use via keyboard controls.
  • Form elements and user input controls should be easily identifiable by a screen reader and easy to navigate using keyboard controls.
  • Visual feedback should be described and accessible by a screen reader. Examples include loading states, notifications and status updates, animations and control state changes such as focus/active or hover.

Motor, Neurological or Cognitive

As with severe visual impairments, motor, neurological and cognitive impairments can have many challenges for accessibility. Below the following should be considered in any case:

  • Navigation should be consistent with other instances and easy to understand.
  • Controls and inputs should not require precision to activate/focus or require a mouse exclusively. Each control should be keyboard accessible. Refer to the section below Keyboard Control for more information.
